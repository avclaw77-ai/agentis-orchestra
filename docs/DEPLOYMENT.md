# Deployment Guide

## Prerequisites

- Docker Engine 24+ and Docker Compose V2
- A domain name pointed to your server (for SSL)
- At least one model provider: Claude CLI (Pro subscription) or an API key (Anthropic, OpenRouter, OpenAI)

## Quick Start

```bash
git clone https://github.com/AgentisLab/agentis-orchestra.git
cd agentis-orchestra
make setup    # generates .env with random secrets
make up       # starts all services
```

Open `http://localhost:3000` and follow the setup wizard.

## VPS Deployment (DigitalOcean / Hetzner / Hostinger)

### 1. Provision a server

- **Minimum**: 2 vCPU, 4 GB RAM, 40 GB SSD
- **Recommended**: 4 vCPU, 8 GB RAM, 80 GB SSD
- Ubuntu 22.04 or 24.04 LTS
- Open ports: 80, 443

### 2. Install Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Log out and back in
```

### 3. Clone and configure

```bash
cd /opt
git clone https://github.com/AgentisLab/agentis-orchestra.git
cd agentis-orchestra
make setup
```

Edit `.env` to set your domain and any API keys:

```bash
nano .env
# Set DOMAIN=orchestra.yourcompany.com
# Add API keys as needed
```

### 4. Start with production compose

```bash
docker compose -f docker-compose.prod.yml up -d
```

Caddy automatically provisions SSL certificates via Let's Encrypt. Your instance will be live at `https://your-domain.com` within a minute.

### 5. Verify

```bash
make health
```

## SSL Setup

The production compose includes Caddy as a reverse proxy. It handles SSL automatically.

**Requirements:**
- Port 80 and 443 open on your firewall
- DNS A record pointing to your server IP
- `DOMAIN` set in `.env`

That's it. Caddy handles certificate provisioning, renewal, and HTTP-to-HTTPS redirects.

**Custom SSL (bring your own cert):**

Edit the `Caddyfile`:

```
orchestra.yourcompany.com {
    tls /path/to/cert.pem /path/to/key.pem
    reverse_proxy app:3000
}
```

## Backup Strategy

### Automated daily backups

```bash
# Add to crontab (runs at 3 AM daily)
crontab -e
0 3 * * * cd /opt/agentis-orchestra && ./scripts/backup.sh >> /var/log/orchestra-backup.log 2>&1
```

Backups are gzipped SQL dumps saved to `./backups/`. Old backups are auto-pruned after 30 days.

### Manual backup

```bash
make backup
```

### Off-site backup

Sync backups to remote storage:

```bash
# S3
aws s3 sync ./backups/ s3://your-bucket/orchestra-backups/

# Rsync to another server
rsync -az ./backups/ user@backup-host:/backups/orchestra/
```

### Restore from backup

```bash
gunzip -c backups/agentis_orchestra_20260415-030000.sql.gz | \
  docker compose exec -T db psql -U agentis agentis_orchestra
```

## Monitoring

### Health checks

All services have built-in Docker health checks. Run a manual check:

```bash
make health
```

### Cron-based monitoring

```bash
# Check every 5 minutes, alert on failure
*/5 * * * * cd /opt/agentis-orchestra && ./scripts/healthcheck.sh || curl -s https://your-alerting-webhook
```

### Logs

```bash
make logs           # All services
make logs-app       # App only
make logs-bridge    # Bridge only
make logs-db        # Database only
```

Log rotation is configured in the production compose (20 MB max, 5 files retained per service).

## Updating

```bash
cd /opt/agentis-orchestra
git pull

# Rebuild and restart
docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up -d

# Verify
make health
```

If using pre-built images from GHCR:

```bash
docker compose -f docker-compose.prod.yml pull
docker compose -f docker-compose.prod.yml up -d
```

## Environment Variables Reference

| Variable | Required | Description |
|----------|----------|-------------|
| `DOMAIN` | Yes (prod) | Your domain for SSL (e.g., `orchestra.company.com`) |
| `DB_PASSWORD` | Yes | PostgreSQL password |
| `BRIDGE_TOKEN` | Yes | Shared secret between app and bridge |
| `ENCRYPTION_KEY` | Yes | 32-byte hex key for encrypting provider API keys |
| `NEXTAUTH_SECRET` | Yes | Session signing key |
| `ADAPTER_MODE` | No | `cli`, `sdk`, or `api` (default: `cli`) |
| `ANTHROPIC_API_KEY` | No | For SDK/API adapter mode |
| `OPENROUTER_API_KEY` | No | Access to 100+ models via OpenRouter |
| `OPENAI_API_KEY` | No | Direct OpenAI API access |
| `PERPLEXITY_API_KEY` | No | Web search with citations |

All secrets are auto-generated by `make setup`. Only add API keys for the providers you want to use.

## Troubleshooting

**Services won't start:**
```bash
docker compose -f docker-compose.prod.yml logs
```

**Database connection errors:**
```bash
docker compose -f docker-compose.prod.yml exec db pg_isready -U agentis
```

**SSL certificate issues:**
- Ensure ports 80/443 are open
- Ensure DNS is pointed to your server IP
- Check Caddy logs: `docker compose -f docker-compose.prod.yml logs caddy`

**Out of memory:**
- Check resource usage: `docker stats`
- Increase server RAM or adjust limits in `docker-compose.prod.yml`
