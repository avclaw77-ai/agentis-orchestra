## WORKSTREAM 1: v1.1 Release Plan

### v1.1 Core Features (2-week sprint)

#### Feature 1: Simulation Gym (5 days)
Test agent personas before deploying them to production. "No blind deployments."

**What to build:**
- New modal in agent profile Soul tab: "Test Persona"
- Accepts current persona + test prompt (e.g., "Handle a customer complaint")
- Runs 3-5 simulated responses via cheapest model
- Shows output variance + confidence score
- Edit-and-retry without saving to production
- Reuses existing `executeWithProvider()` infrastructure

**Files:**
- `app/src/components/simulation-gym.tsx` -- new component
- `app/src/app/api/agents/[id]/simulate/route.ts` -- new API route
- `app/src/components/agent-profile.tsx` -- add "Test" button in Soul tab
- `bridge/server.ts` -- add `/agents/:id/simulate` endpoint

**Why:** Salesforce identified simulation gyms as a 2026 trend. We have the execution pipeline -- just need the UX. Reduces bad deployments.

**Why:** Enterprises want proof that agents improve over time. This makes Soul Engine's value visible.

## WORKSTREAM 2: AgentisLab.ai Website Overhaul

### Audit Summary (UIUX team rating: 6.5/10)

**What's working:**
- Copy is genuinely strong ("We don't advise on AI. We build it and ship it.")
- Bilingual support is solid
- Service descriptions are specific and credible

**What's broken (AI slop patterns):**

1. **Decorative animations everywhere**: Pond ripples, fish elements, floating badges, pulsing dots, rotating 50x badge. 10+ infinite animations running simultaneously. Zero UX purpose.

2. **Generic SaaS template feel**: Gradient text headings, centered everything, -10px lift-on-hover, frosted glass navbar, white-circle icons. This is the 2023 SaaS playbook.

3. **Fake UI mockups**: DevStudio section shows a window with fake data and a non-existent URL (devstudio.agentislab.ai). Undermines credibility.

4. **Typography uniformity**: Body text is all Inter weight 300-500, no emphasis variety. Serif font used inconsistently for quotes only.

5. **Color system leaking**: Only 3 brand colors defined but 10+ Tailwind defaults used. No semantic system.

6. **Copy-paste bug**: "our AI pipeline Pipeline" appears twice in translations.

7. **Accessibility gaps**: No focus-visible states, no prefers-reduced-motion, low contrast on trust pills, empty href="#" links.

8. **Orchestra section**: Dark bg in middle of light page, tiny text, white-on-blue contrast issues (fixed but needs further review).

### Design Overhaul Plan

#### Phase 1: Remove the slop (Day 1)
- Kill all decorative animations (ripples, fish, floating badges, pulsing dots)
- Remove the DevStudio section entirely (product doesn't exist publicly)
- Fix "our AI pipeline Pipeline" bug
- Remove empty href="#" links
- Add `prefers-reduced-motion` media query

#### Phase 2: Typography + Color system (Day 2)
- Establish weight variety: 300 for descriptions, 500 for body, 700 for labels, 800 for headlines
- Define full semantic color palette in Tailwind config (not just 3 brand colors)
- Fix trust pill contrast (10px text-slate-500 -> 12px text-slate-700)
- Smooth typography scaling (add intermediate breakpoints or use clamp())

#### Phase 3: Section redesign (Days 3-4)
- **Hero**: Remove gradient text, use bold weight instead. Remove floating badges. Keep the headline copy.
- **Services**: Asymmetric card layout instead of centered 3-column grid. Remove white-circle icons.
- **Results**: Fix accordion keyboard accessibility. Remove pulsing status dots.
- **Team**: Remove bouncing 50x badge animation. Fix founder photo ring styling. Remove serif quotes.
- **Process**: Remove fake progress bar animation. Balance left/right layout weight.
- **Contact**: Add real social links or remove icon placeholders.
- **Footer**: Increase trust pill contrast and font size.

#### Phase 4: Accessibility + Polish (Day 5)
- Add focus-visible states to all interactive elements
- Add aria-expanded to accordion cards
- Verify WCAG AA contrast on all text
- Test responsive at 320px, 768px, 1024px, 1440px
